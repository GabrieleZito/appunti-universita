Problema computazionale → relazione che associa un input ad un output

Algoritmo → dato un problema computazionale, sequenza finita di passi elementari e non ambigui  che trova una soluzione al problema computazionale

- Esempio 1
    
    Calcolare il minimo di un insieme $S$
    
    Definizione formale:
    
    $$
    \min(S)=a \Leftrightarrow\ \exists\ a\in S:\forall\ b\in S,\ a\leq b
    $$
    
- Esempio 2
    
    Sia $S=[S_1, S_2,\dots,S_n]$ sequenza ordinata e distinta
    
    Ricercare la posizione di un dato $v$ in $S$ (restituire l’indice se presente, 0 altrimenti)
    
    Definizione formale:
    
    $$
    find(v,S)=\begin{cases}
    i\quad \text{if}\ \exists\ i\in[1,\dots,n]:S_i=v\\
    0\quad \text{else}
    \end{cases}
    $$
    

## Pseudo codice

Permette di scrivere un algoritmo in modo formale, scegliendo il livello di dettaglio, a prescindere la linguaggio di programmazione usato.

- Esempi
    
    Elemento minimo
    
    ```python
    function MIN(S,n)
    	for i=1 to n do
    		isMin = true
    		for j=1 to n do
    			if i != j and S[j]<S[i] then
    				isMin = false
    		if isMin = true then
    			return S[i]
    ```
    
    Ricerca
    
    ```python
    function FIND(S,n,v)
    	for i=1 to n do
    		if S[i]=v then
    			return i
    	return 0
    ```

## Ricerca binaria

$S$  insieme di elementi ordinati

Si prende l’elemento centrale $S[m]$:

1. $S[m]=v$ → elemento trovato
2. $v<S[m]$ → cerco a sinistra di $S[m]$
3. $v>S[m]$ → cerco a destra di $S[m]$

### Pseudocodice

$\text{function}\ \text{\color{cyan}{BinarySearch}}(S,v,first, last)$
$\quad \text{If}\ first >last\ \text{then}$
$\qquad \text{return }0$
$\quad m = \lfloor\frac{first+last}{2}\rfloor$
$\quad\text{If}\ S[m]=v\ \text{then}$
$\qquad \text{return }m$
$\quad \text{else if}\ S[m]>v\ \text{then}$
$\qquad\text{return}\ \text{\color{cyan}BinarySearch}(S,v,first,m-1)$
$\quad\text{else}$
$\qquad\text{return}\ \text{\color{cyan}BinarySearch}(S,v,m+1,last)$

## Valutazione della correttezza

Invariante → Condizione vera in un certo punto dell’algoritmo

Invariante di ciclo → Condizione vera all’inizio dell’iterazione di un ciclo

Invariante di classe → Condizione vera al termine dell’esecuzione di un metodo di una classe

Invariante di ciclo: usato per gli algoritmi iterativi

3 fasi:

1. Inizializzazione (caso base): condizione vera all’inizio dell’iterazione 
2. Conservazione (passo induttivo): se la condizione è vera all’inizio dell’iterazione, lo rimane alla sua fine
3. Conclusione: quando il ciclo termine l’invariante rappresenta la correttezza dell’algoritmo
