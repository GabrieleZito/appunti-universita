Query

```sql
SELECT ListaAttributi
FROM ListaTabelle
WHERE condizioni
```

Alias

```sql
SELECT nome AS N
SELECT nome N
```

Condizioni

- Operatori di confronto: $=,<,<=,>,>=, <>$
- Connettori logici: AND, OR, NOT
- Operatori:   BETWEEN, IN, LIKE, IS NULL
    
    ```sql
    WHERE attr BETWEEN 1000 AND 2000
    WHERE attr IN ("prova","prova1")
    WHERE attr LIKE "%G_"      // % -> 0+ caratteri, _ -> 1 carattere
    WHERE attr LIKE "C#_F" ESCAPE "#" // per definire un carattere di escape 
    ```
    

Versioni negate: NOT BETWEEN, NOT IN, NOT NULL, NOT LIKE

Nella lista attributi del select si possono anche effettuare operazioni aritmetiche tra colonne

### Ordinamento

```sql
SELECT ListaAttributi FROM ListaTabelle
WHERE condizioni ORDER BY [ASC/DESC]
```

ASC è l’ordinamento di default

## Operatori aggregati

### Count

- COUNT(*) → conta il numero di righe della tabella
- COUNT(ALL attr.) → conta il numero di valori non nulli dell’attributo
- COUNT(DISTINCT attr.) → conta il numero di valori non nulli distinti dell’attributo

### Max(attr.)/Min(attr.) → calcola il massimo/minimo valore di una colonna

### Sum

- SUM(ALL attr.) → somma tutti i valori non nulli
- SUM(DISTINCT attr.) → somma tutti i valori non nulli distinti

### Avg

- AVG(ALL attr.) → calcola la media dei valori della colonna
- AVG(DISTINCT attr.) → calcola la media dei valori della colonna ignorando i duplicati

Per tutti gli operatori, ALL è il default, quindi può anche essere omesso

Quando si usa un operatore aggregato non può essere mischiato con altri attributi semplici. O si usano solo attributi, o solo operatori aggregati, eccetto se si usa il GROUP BY

### GROUP BY

```sql
SELECT Dipartimento, AVG(Stipendio)
FROM Impiegati
GROUP BY Dipartimento
```

Si usa per raggruppare le righe per valore di colonna.

Abbiamo potuto mettere il Dipartimento insieme al AVG perché abbiamo raggruppato per lo stesso Dipartimento.

Si possono imporre delle condizioni sull’operatore aggregato con:

```sql
SELECT Dipartimento, AVG(Stipendio)
FROM Impiegati
GROUP BY Dipartimento
HAVING AVG(Stipendio) > 1200
```

### INSERT

```sql
INSERT INTO tabella(listaAttributi) VALUES(listaValori)
```

Inserimento di una nuova riga

La lista attributi è opzionale se si inseriscono tutte le colonne (l’ordine è importante)

### DELETE

```sql
DELETE FROM tabella WHERE condizione
```

Elimina le righe che rispettano una data condizione

Se la condizione è omessa elimina tutto

### UPDATE

```sql
UPDATE tabella SET attr=valore WHERE condizione
```

## JOIN

Il join si effettua imponendo nel WHERE la condizione per il collegamento delle tabelle

```sql
SELECT attributi FROM tab1, tab2 WHERE tab1.attr1=tab2.attr2
SELECT attributi FROM tab1, tab2 ON tab1.attr1=tab2.attr2
```

### Inner join

Join effettuato come sopra ma in cui la condizione non è necessariamente di uguaglianza

### Cross join

```sql
SELECT attributi FROM tab1, tab2
```

Senza una condizione viene effettuato un prodotto cartesiano

### Natural join

Join creato sulle colonne che hanno lo stesso nome

```sql
SELECT attributi FROM tab1 NATURAL JOIN tab2
```

Se ci sono più colonne uguali e vogliamo specificare quali considerare:

```sql
SELECT attributi FROM tab1 JOIN tab2 USING (listaAttributi)
```

### Outer join

```sql
SELECT attributi FROM tab1 LEFT JOIN tab2
SELECT attributi FROM tab1 RIGHT JOIN tab2
SELECT attributi FROM tab1 FULL JOIN tab2
```

usato per mantenere i valori di una o entrambe le tabelle anche se non fanno parte del join

## Subquery

Comando SELECT usato all’interno di un’altra query

Possono essere usate per:

- espressioni di confronto
- espressioni di confronto quantificate
- espressioni IN
- espressioni EXISTS
- calcolo di espressioni

### Tipi di subquery

- Scalare → select che restituisce un solo valore
- Di colonna → select che restituisce una colonna
- Di tabella → select che restituisce una tabella

### Subquery in espressioni di confronto

- La subquery deve essere scalare
- la subquery deve essere inserita DOPO l’operatore
- non si può fare un confronto tra due subquery
- nella subquery non si possono usare HAVING e GROUP BY

```sql
SELECT * FROM Veicoli
WHERE categoria = (SELECT cod_cat FROM Categorie WHERE nome_cat = "autovettura")
```

### Subquery in espressioni di confronto quantificato

- la subquery deve essere di colonna
- la subquery dopo l’operatore
- no confronto tra due subquery
- no HAVING o GROUP BY nella subquery

Si effettuano usando gli operatori di confronto quantificato:

- ANY → vero se almeno uno dei valori della subquery soddisfa la condizione
- ALL → vero se tutti i valori della subquery rispettano la condizione
- EXISTS → vero se la subquery restituisce almeno una tupla

Dal punto di vista logico

- $=ANY$ è equivalente a $IN$
- $<>ALL$  è equivalente a $NOT\ IN$

```sql
SELECT nome_cat FROM Categorie
WHERE EXISTS (SELECT * FROM Veicoli WHERE Categorie.cod_cat=Veicoli.categoria) 
```

La subquery può accedere alle variabili della query principale

Nel caso dell’EXISTS il confronto tra la tabella esterna e quella interna è necessario, se no non funzionerebbe.

Si può vedere come due for innestati

### Subquery nel calcolo di espressioni

```sql
SELECT Targa, Cilindrata, Cilindrata-(SELECT MIN(Cilindrata) FROM Veicoli) Differenza
FROM Veicoli
```

### Subquery nidificate

È possibile usare più subquery, anche una dentro l’altra, ma ricordandosi che una subquery vede solo le variabili della query superiore

## Operatori Booleani

### Union

Restituisce l’unione delle righe di più tabelle

```sql
SELECT Padre, Figlio  //la tab risultante avrà le colonne con il nome della prima tabella
FROM Paternita
UNION [ALL]
SELECT Madre, Figlio  //l'ordine degli attributi è importante
FROM Maternita
```

### Except

Differenza tra tabelle

```sql
SELECT ---
EXCEPT [ALL]
SELECT ---
```

### Intersect

Intersezione tra tabelle

```sql
SELECT ---
INTERSECT [ALL]
SELECT ---
```
