Un trigger definisce un’azione che il DB deve compiere al verificarsi di un certo evento.

Tipicamente usati per gestire vincoli di integrità, calcolare dati derivati, gestire eccezioni, mantenere la consistenza dei dati ridondanti, etc..

È bene che sia il DBMS ad occuparsi di garantire la consistenza dei dati, e non l’applicazione, e i trigger ce lo permettono.

Per poter creare un trigger su una tabella: si deve esserne proprietari e bisogna avere il privilegio ALTER e CREATE INDEX

### Tipi di Trigger

- Livello riga → eseguiti una volta per ciascuna riga modificata in una transazione. Clausola FOR EACH ROW
- Livello istruzione → eseguiti una volta per transazione
- Before/After → eseguiti prima o dopo l’operazione specificata
- Instead Of → per specificare cosa fare invece di eseguire le operazioni che hanno attivato il trigger

### Sintassi

```sql
CREATE TRIGGER nome
BEFORE/AFTER {INSERT, UPDATE, DELETE}
ON Tabella
[FOR EACH ROW]
WHEN condizione_che_attiva_il_trigger
codice_da_eseguire
```

### Before Trigger

```sql
CREATE TRIGGER verifica_Sal_Imp
BEFORE INSERT ON Impiegati
FOR EACH ROW
WHEN (New.Salario > (SELECT Salario FROM Impiegato WHERE CodiceImp = New.Cap))
SIGNAL SQLSTATE"70000"("Stipendio troppo elevato")
```

Il trigger verifica che lo stipendio di un impiegato non sia più alto del suo capo

SIGNAL serve a segnalare un errore e annulla gli effetti dell’evento che ha attivato il trigger

I BEFORE Trigger vengono usati per far rispettare i vincoli di integrità oppure quei vincoli  persi nella traduzione da schema concettuale a relazionale

Operazioni possibili in un BEFORE Trigger:

- INSERT
- SIGNAL
- SET (modifica 1+ campi della tupla nuova della tabella)

### After Trigger

```sql
CREATE TRIGGER UpdateQtaResidua
AFTER INSERT ON Vendite
FOR EACH ROW
UPDATE Giacenze
SET QtaResidua = QtaResidua - new.Quantita
WHERE Giacenze.codProdotto = new.codProdotto
```

Utili per calcolare dati derivati

Operazioni possibili:

- SELECT
- INSERT
- DELETE
- UPDATE
- SIGNAL

### Eliminazione Trigger

```sql
DROP TRIGGER nome
```

### Combinazione Trigger

Si possono specificare più operazioni che attivano il trigger

```sql
BEFORE INSERT OR UPDATE
```

Nel corpo del trigger, per specificare a quale caso ci stiamo riferendo:

```sql
IF INSERTING/UPDATING/DELETING
...
ELSE
```

---

Trigger Attivo → trigger che modifica lo stato della base di dati

Trigger Passivo → trigger che serve a provocare fallimenti della transazione sotto certe condizioni
