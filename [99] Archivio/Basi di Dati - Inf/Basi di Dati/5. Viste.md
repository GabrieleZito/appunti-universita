Sono delle tabelle virtuali, i cui dati sono riaggregazioni dei dati delle tabelle fisiche

Forniscono una visione esterna del DB agli utenti. Possono esserci viste ad accesso limitato per determinati utenti

### Limitazioni

- No Union, Intercept, Except
- No Order By

### Creazione

```sql
CREATE VIEW nome[(listaAttributi)] AS selectQuery
[WITH [LOCAL | CASCADED] CHECK OPTION]
```

La lista attributi viene usata per rinominare le colonne

L’opzione WITH CHECK OPTION assicura che per le operazioni di inserimento e modifica vengono rispettate le condizioni definite nel WHERE della vista stessa

- LOCAL → verifica le condizioni solo della propria vista
- CASCADED → verifica anche le condizioni di eventuali viste annidate

Il contenuto di una vista è dinamico, ma la struttura no. Se in una tabella di una vista viene modificata, aggiunta o eliminata una colonna, questa modifica non si riflette nella vista  

### Eliminazione

```sql
DROP VIEW nome RESTRICT/CASCADE
```

- Restrict → viene eliminata solo se non dipende da altri oggetti
- Cascade → elimina la vista e tutte le dipendenze

### Viste di gruppo

Vista in cui viene usato un operatore aggregato. In questo caso è obbligatorio rinominare la colonna.

### Viste aggiornabili

Una vista è aggiornabile se è possibile risalire al dato “puro” nelle tabelle di riferimento, quindi:

- Non utilizzare DISTINCT
- Nel FROM è presente una sola tabella, o vista a sua volta aggiornabile
- Non sono presenti espressioni o operatori aggregati
- Nel WHERE non c’è una subquery
- Non c’è GROUP BY o HAVING
