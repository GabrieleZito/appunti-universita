### Divisione in SQL

Fornitore(Fid, Fnome, Indirizzo)

Pezzo(Pid, Pnome, Colore)

Catalogo(CodF, CodP, Costo)

FK: CodF REFERENCES Fornitore(Fid)

FK: CodP REFERENCES Pezzo(Pid)

```sql
SELECT Pid, Pnome FROM Pezzo P
WHERE not exists(SELECT * FROM Fornitore F
									WHERE not exists (SELECT * FROM Catalogo C
																		WHERE C.CodF=F.Fid AND C.CodP=P.Pid))
```

### Vincoli

UNIQUE(X) → crea un vincolo di superchiave sull'insieme di attributi X

FOREIGN KEY (X) REFERENCES Tab(Y) → crea un vincolo di integrità referenziale tra gli attributi X e gli attributi Y della tabella Tab.

### Creare un dominio

In SQL si può creare un sottodominio di un dominio di base, 

```sql
CREATE DOMAIN Voto
AS SMALLINT DEFAULT NULL
CHECK (value >= 18 AND value <= 30)
```

### Controlli sulle tabelle

### Check

Vincolo che viene controllato ogni volta che viene inserita una tupla. 

Viene inserita la tupla → verifica della condizione → se non è rispettata viene eliminata la tupla

```sql
CREATE TABLE tabella(
Attributo check(condizione)
...
check(condizione)
)
```

Il primo check è un controllo sul dominio di un attributo

Il secondo check è un controllo sulle tuple della tabella

### Asserzioni

Quando si deve imporre un vincolo che comprende più di una tabella, non si può più usare il check, ma si usano le asserzioni, cioè dei vincoli sul database.

```sql
CREATE ASSERTION Nome
	CHECK (condizione)
```

### Vincoli immediati

Vincoli che vengono verificati dopo ogni operazione che coinvolge le tabelle presenti nel vincolo.

Vincoli immediati sono DEFAULT, PRIMARY KEY, FOREIGN KEY, UNIQUE

Quando un vincolo immediato non viene rispettato viene annullata l'operazione (Rollback parziale)

### Vincoli differiti

Vincoli che vengono verificati al termine di una serie di operazioni (transazione)
