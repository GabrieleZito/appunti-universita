La programmazione asincrona riguarda la possibilitàdi lanciare da programma più thread di lavoro paralleli e gestire i loro risultati

è utile quando c’è un carico computazionale molto alto o durante la gestione delle richieste HTTP asincrone verso il server

In JavaScript esistono diversi modi per innescare una esecuzione asincrona

- Web Worker
- Creazione e gestione di *Promise*
- Richieste asincrone al server con fetch o creazione di *XMLHTTPRequest*

### Web Worker

Processo separato lanciato dal programma principale incorporato nella pagina HTML del client che comunica con quest’ultimo tramite scambio di messaggi

La classe Worker fa parte della Web API di Javascript

```jsx
<script>
	//codice ddello script
	let worker = new Worker('fileJSdelWorker')
	
	worker.addEventListener('message',function(event){
		//callback che gestisce le risposte del worker che si trovano in event.data
	})
	worker.postMessage(oggettoMessaggioDati)
</script>
```

```jsx
//codice del worker
onMessage = function(event){
	//callback che gestisce le operazioni del worker quando riceve un messaggio il cui contenuto si trova in event.data
	postMessage(oggettoMessaggioRisposta)
}
```

### Promise

La classe Promise definisce una operazione differita che viene eseguita asincronamente rispetto al resto del programma

Può trovarsi in 3 stati:

- pendente
- risolta - eseguita correttamente
- rigettata - esecuzione fallita
