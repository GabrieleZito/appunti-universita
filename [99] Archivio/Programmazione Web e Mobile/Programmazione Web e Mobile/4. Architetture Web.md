
Il WWW è anche utilizzato per accedere a delle informazioni, e non solo a degli ipertesti

In questo caso c’è la necessità di utilizzare delle pagine web dinamiche il cui contenuto può cambiare e che dipende dalle azioni svolte dall’utente.

L’elaborazione di queste pagine può avvenire sul client o sul server, o entrambi.

### Elaborazione Client-Side

- Pagine HTML con codice sorgente che viene eseguito direttamente dal browser quando vengono ricevute
- Javascript, VBScript (linguaggi di scripting)

### Elaborazione Server-Side

- Vengono eseguiti dei programmi sul server web, invocati attraverso appositi URL e generano una pagina HTML da mandare al client
    
    Interfaccia CGI - Common Gateway Interface - è la più antica tecnologia per il passaggio dei dati ai programmi sul server
    

ASP → Active Server Pages - consente di utilizzare i linguaggi di scripting sul server. Codice delimitato da `<% ... %>`. 

PHP → Il codice viene elaborato sul server e in output mandano codice HTML al client 

### Architettura LAMP

Linux, Apache, MySQL, PHP

### Applet e Servlet

- Applet - riferita all’interno di una pagina con il tag `<object>`. L’effettiva applicazione viene spedita separatamente al client. Ha molte limitazioni per questione di sicurezza. Praticamente non esistono più
- Servlet o *web application* - applicazione server scritta in java. Sono gestite da dei *application server* che estendono le funzionalità di un web server

### JSP

Tecnologia per costruire app web direttamente da HTML, con tag specifici che contengono codice Java

### Modello Three-Tier

- Presentation Tier - interfaccia verso tutti i dispositivi
- Application/Business Logic Tier - cuore dell’applicazione, include il server web e le tecnologie correlate
- Data Tier - Insieme di archivi di dati

### Pattern MVC

- Controller
- View
- Model
