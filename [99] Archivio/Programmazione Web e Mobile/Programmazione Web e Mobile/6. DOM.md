### Document Object Model

interfaccia che permette di accedere e modificare un documento

- L’intero documento è un nodo documento
- Ogni tag è un nodo elemento
- Il testo contenuto in un tag è un nodo testo
- Ogni attributo XML è un nodo attributo
- I commenti sono nodi commento

Ognuno di questi nodi fa parte dell’albero documento con radice il nodo documento

### W3C DOM

API per accedere e manipolare documenti HTML e XML - Oggetto document e classsi Node e Element

Diviso in 3 parti:

- Core DOM - definisce un insieme standard di oggetti per qualunque documento
- XML DOM - definisce un insieme standard di oggetti per i documenti XML
- HTML DOM - definisce un insieme standard di oggetti per i documenti HTML

Gli oggetti dell’API si dividono in 4 livelli:

- DOM Level 1 - specifiche base per Core, HTML e XML
- DOM Level 2 - specifiche relative alla gestione degli eventi, dei namespace XML, l’accesso dinamico e l’attraversamento dell’albero
- DOM Level 3 - specifiche relative a validazione ,serializzazione e visita del documento utilizzando XPath
- DOM Level 4 - “living standard” del WHATWG

### Document Type Definition

- Può essere all’inizio del documento
    
    ```xml
    <!DOCTYPE prova [
    	<!ELEMENT tag (#PCDATA)>   // PCDATA -> testo
    ]>
    ```
    
- Può fare riferimento a file esterni
    
    ```xml
    <!DOCTYPE prova SYSTEM "myDTD.dtd" [
    	<ELEMENT tag (#PCDATA)>
    ]>
    ```
    
    oppure
    
    ```xml
    <!DOCTYPE HTML PUBLIC "-//W3C//DTD HMTL 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
    ```
    
    prefisso//proprietario//nome DTD//lingua
    

### Elementi

```xml
<!ELEMENT nomeTag (tagFiglio)>
<!ELEMENT nomeTag (tagFiglio, tagFiglio2)> //contiene tagFiglio e tagFiglio2
<!ELEMENT nomeTag (tagFiglio|tagFiglio2)>  //contiene tagFiglio o tagFiglio2
<!ELEMENT nomeTag (tagFiglio+)>            //contiene uno o più tagFiglio
<!ELEMENT nomeTag (tagFiglio*)>            //contiene zero o più tagFiglio
<!ELEMENT nomeTag (tagFiglio?)>            //contiene zero o un tagFiglio
<!ELEMENT nomeTag EMPTY>                   //non contiene niente
<!ELEMENT nomeTag ANY>                     //contenuto misto (PCDATA, altri elementi, o entrambi)  
```

### Attributi

```xml
<!ELEMENT tag EMPTY>
<!ATTLIST nomeTag nomeAtt proprietà>

proprietà:
CDATA -> il valore può contenere qualunque carattere
#REQUIRED -> attributo obbligatorio
#IMPLIED -> non obbligatorio
#FIXED "..." -> valore fissato
```

Tipi di attibuti

Stringhe

```xml
<!ELEMENT tag EMPTY>
<!ATTLIST nomeTag nomeAtt CDATA>
```

Enumerati

```xml
<!ELEMENT tag EMPTY>
<!ATTLIST nomeTag nomeAtt (M|F) "F">   //può assumere solo il valore M o F, F valore di default
```

Token

- ID - Identificatore unico
- IDREF - riferimento a identificatore unico
- IDREFS - lista di riferimenti as identificatori
- ENTITY - entità
- ENTITIES - lista di entità
- NMTOKEN - nome XML valido
- NMTOKENS - lista di nomi XML
- NOTATION - DTD Notation, riferimento ad entità esterna non XML
- xml:lang / xml:space - attributi predefiniti XML

### Notation

Una Notation definisce il formato dati di una entità *unparsed,* cioè che non fa riferimento a dati XML

Anche queste possono essere SYSTEM o PUBLIC

```xml
<!NOTATION jpg PUBLIC "JPG 1.0">
<!NOTATION png SYSTEM "image/png">
```
