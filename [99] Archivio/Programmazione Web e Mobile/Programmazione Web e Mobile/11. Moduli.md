In Javascript un progetto può essere diviso in pacchetti, che a loro volta contengono moduli

Pacchetto → componente software dedicato ad assolvere una certa categoria di compiti, autoconsistente, che può essere distribuito ed installato autonomamente

- é articolato in uno o più moduli
- può dipendere da altri pacchetti o essere una dipendenza
- viene aggiornato e manutenuto autonomamente

Modulo → unità di codice che espone all’esterno una data interfaccia e nasconde completamente l’implementazione interna

Un progetto software dipende in genere da molti pacchetti che a loro volta hanno le loro dipendenze

### Node Package Manager

- Upload di pacchetti nuovi o aggiornati
- Download e installazione di pacchetti disponibili
- Gestione delle differenti versioni dei pacchetti “dipendenti”

### Moduli CommonJS

Standard utilizzato da NodeJS

Utilizzano un oggetto chiamato *exports* per esportare le proprietà/metodi di interfaccia

### Moduli ECMSAScript

Definiti da ES rev 6 in poi

Si basano su *export* e *import*

Non è possibile richiamare moduli ES al di fuori di altri moduli ES

L’esecuzione di un’applicazione che importa moduli ES necessita esplicitamente che l’ambiente di esecuzione sia etichettato come modulo

- `<script type=’module’></script>`
- nel file package.json di node definire l’attributo “type: ‘module’”
