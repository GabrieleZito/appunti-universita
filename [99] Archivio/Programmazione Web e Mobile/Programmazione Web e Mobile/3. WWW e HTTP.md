## World Wide Web

Servizio di consultazione di documenti organizzati secondo una struttura ipertestuale distribuita sulla rete internet

Ipertesto → organizzazione di materiale informativo in forma tale da consentirne una fruizione non lineare in cui sono presenti dei percorsi di fruizione predefiniti secondo un preciso progetto comunicativo.

Sito web → un particolare tipo di ipertesto

### Elementi di un ipertesto

Nodo → unità informativa autosufficiente, testuale e non.

Ancora → punto di origine o termine di un collegamento ad un’altra unità

Link → collegamento ad un’altra unità, può avere diverse funzioni ed è agganciato a due ancore

### Architettura dell’informazione

Consiste nel design strutturale di un sito web che ne facilita l’esecuzione dei compiti e l’accesso ai contenuti

Si può definire come la combinazione di organizzazione, categorizzazione e schemi di navigazione

### Risorse informative

URI → Uniform Resource Identifier; stringa che identifica univocamente una risorsa generica sulla rete

URN → Uniform Resource Name; URI che identifica una risorsa mediante un nome appartenente ad un particolare dominiodi nomi (namespace). Non da indicazioni sulla sua esistenza e/o posizione in rete

Sintassi: urn: `<Namespace IDentifier>:<Namespace Specific String>`

Es. :   urn:isbn:0-395-36341-1

URL → Uniform Resource Locator; URI che identifica una risorsa e che inoltre permette una sua rappresentazione descrivendo il suo meccanismo di accesso primario

Sintassi: `<protocollo><indirizzo><percorso>`

Es. : http://www.unipa.it/persone/docenti 

### Tipi di URL

- Richiesta di un documento ipertestuale:
    
    `http://<nome_server>[:<numero_porta>]/<percorso>[/<nome_file_html>][#<ancora>]`
    
    - Numero porta: la porta TCP di erogazione del servizio, se diversa da 80
    - Se il nome del file html manca, si assume index.html, default.html o index.php, altrimenti va specificato
    - Ancora: riferimento ad una zone precisa all’interno della pagina

- Richiesta di trasferimento file sicuro:
    
    `sftp://[<username>[:<password>]@]<nome_server>/<percorso>/<nome_file>`
    
    - Username e password: identificazione di un utente specifico, altrimenti si assume l’utente anonymous senza password

- Richiesta di terminale remoto sicuro:
    
    `ssh://<utente>[:<password>]@<nome_server>[:<numero_porta>]`
    
- Richiesta di invio posta elettronica:
    
    `mailto:<nome_utente>@<nome_server>`
    

## HTTP

Protocollo di applicazione TCP (Hyper-Text Transfer Protocol) con un modello client-server

La comunicazione avviene attraverso coppie richiesta-risposta su stream TCP

### HTTP Request

![[www5.png]]
- Linea di richiesta
    - Metodi GET e POST
- Linee di intestazione
    - Host: URL che stiamo chiedendo
    - User-agent: Identificazione del browser/SO/piattaforma software
    - Accept: Formati dei dati, compressione, linguaggio e set di caratteri accettati dal browser
    - Keep-Alive, Connection: Tempo di mantenimento della connessione che viene impostata come [persistente](3.%20WWW%20e%20HTTP.md) per quel lasso di tempo
    - Referer: L’URL da cui il browser sta eseguendo la richiesta

### HTTP Response

![Untitled](www8.png)

- Linee di intestazione:
    - Linea di stato: Il codice di stato che informa il client sul soddisfacimento o meno della richiesta, e il motivo   [Codici di Stato HTTP](2.%20Codici%20di%20stato%20HTTP.md)
    - Date: La data della risposta
    - Server: Identificazione server/OS/moduli e liberie installati
    - Last-modified: La data di ultima modifica della risorsa
    - ETag: Entity Tag (Opzionale)usato come chiave della risorsa per la gestione efficiente della [cache](3.%20WWW%20e%20HTTP.md) del browser
    - Accept-Ranges: Il server esaudirà anche richieste parziali di una risorsa inviando gruppi di byte
    - Content-Lenght, Content-Type: Lunghezza in byte e tipo [1. MIME](1.%20MIME.md) della risposta
    - Corpo della risposta: Il flusso di caratteri che compongono la risorsa se HTML, il contenuto binario di un dato multimediale, un file JS o CSS, etc..

- Cache
    
    Uno spazio di archiviazione e di memoria centrale, gestito dal software che la usa, per l’accesso efficiente alle risorse già precaricate e indicizzate.
    
    La cache è usata dai browser, dai server e anche sulla rete attraverso i proxy server.
    
    Direttiva Cache-Control:
    
    - no-store: non conserva in cache
    - no-cache: può conservare in cache (richiede validazione)
    - public: può conservare in cache
    - private: per utenti singoli
    - max-age: scadenza
    - must-revalidate: obbligo di validazione

### Metodi GET e POST

- GET
    
    Richiesta standard si risorse e informazioni
    
    Va bene per richiedere dati, non è sicuro (dati nell’URL)
    
- POST
    
    Usata di norma per inviare informazioni al server e innescare ulteriori elaborazioni
    
    Richiede che i dati siano “subordinati” alla URI richiesta, quindi serve per inviare dati. Non sicuro (informazioni nel corpo dell’HTTP Request). Consente di usare diversi tipi di encoding. Consente di usare una cache delle richieste
    

### URL Encoding

Negli URL non possono essere usati spazie può essere utilizzata anche una tabella di caratteri non standardizzata

La stringa viene ricodificata seguendo il set Unicode UTF8, convertendo gli spazi in + (%20)

### Limitazioni di HTTP

- Protocollo Stateless - Non c’è controllo del tracciamento dell’attività del client, cioè la sessione
- Non ci sono meccanismi interni di sicurezza

### Gestione della sessione

- URL Rewriting - Inserimento dei dati da tracciare nell’URL
- Campi nascosti nei form
- Cookie
    - File di piccole dimensioni conservati dal server nel client
    - Possono essere disabilitati dal client
    - il contenuto è reinviato al server nell’intestazione della risposta
- Sessioni SSL
    - SSL - Secure Socket Layer
    - TLS - Transport Layer Security
    - Strato intermedio tra TCP e HTTP
    - Usa la crittografia RSA a chiave pubblica
    - I server si autenticano con i certificati
